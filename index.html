De la IA a GitHub

Cómo crear una página web con Inteligencia Artificial y publicarla al mundo

🎯 ¿Qué vamos a aprender hoy?

Objetivo: Entender cómo una IA genera código web y cómo llevar ese código a internet usando GitHub.

  1. 🧠 Prompting efectivo: Cómo pedirle a la IA que cree una página web completa.
  2. 📄 Anatomía HTML: Qué hace cada etiqueta en la estructura de la página.
  3. 🎨 Magia CSS: Cómo se convierte el código en algo visual y atractivo.
  4. 🚀 GitHub Pages: Subir tu creación y obtener una URL real.
  5. 🌐 Compartir: Mostrar tu trabajo al mundo (¡y a Homero!).

"¡Woo-hoo! ¡Con IA y GitHub, hasta yo puedo hacer una página web!" — Homero Simpson

🧠 Paso 0: El Prompt (La receta)

Antes de tocar código, le pedimos a la IA que cree la página. Un buen prompt incluye:

# Ejemplo de prompt para la IA "Ayúdame a crear una página web sobre la Copa Mundial de Fútbol 1994 que contenga: - Todos los equipos participantes - Un mapa conceptual del torneo (quién ganó y perdió) - Usar HTML y CSS puro - Que sea visual y fácil de entender"
💡 Tip de Bart: Mientras más específico seas, mejor resultado obtienes. Incluye formato, colores, datos específicos y tipo de interacción.

La IA genera el código HTML completo con estilos CSS embebidos. ¡No necesitas instalar nada!

📄 La anatomía de un archivo HTML

Todo documento HTML tiene una estructura básica obligatoria:

<!DOCTYPE html> <!-- Declara que es HTML5 --> <html lang="es"> <!-- Raíz del documento, idioma español --> <head> <!-- Metadatos: título, estilos, configuración --> <meta charset="UTF-8"> <title>Mi Página</title> <style>...CSS aquí...</style> </head> <body> <!-- Todo lo visible va aquí --> <h1>Hola Springfield</h1> </body> </html>
<head>: El "cerebro" invisible. Contiene título, estilos, fuentes y configuración.
<body>: El "cuerpo" visible. Todo lo que el usuario ve y interactúa.

🏗️ Etiquetas semánticas: La estructura lógica

HTML5 introdujo etiquetas que describen su propósito. En nuestra página del Mundial usamos:

<header> <!-- Encabezado principal: título, nav --> <nav> <!-- Navegación: enlaces internos --> <main> <!-- Contenido principal único de la página --> <section> <!-- Sección temática: equipos, mapa --> <article> <!-- Artículo independiente: cada confederación --> <div> <!-- Contenedor genérico para agrupar --> <h1> a <h3> <!-- Jerarquía de títulos --> <p> <!-- Párrafos de texto --> <ul> / <li> <!-- Listas desordenadas y items -->
🎭 Como diría Lisa: Usar etiquetas semánticas no es solo "ser correcto", ayuda a que lectores de pantalla, motores de búsqueda y tu futuro yo entiendan la estructura.

🧭 El Header: La puerta de entrada

En la página del Mundial, el <header> cumple dos funciones: presentar el título y permitir navegación rápida.

<header> <h1>Copa Mundial de Fútbol 1994 (EE. UU.)</h1> <p>Resumen visual del Mundial...</p> <nav> <a href="#equipos">Selecciones</a> <a href="#mapa-mundial">Mapa del torneo</a> </nav> </header>
¿Qué pasa aquí?
• <h1> es el título principal (solo uno por página).
• <nav> contiene enlaces internos con href="#id" que saltan a secciones.
• <p> describe el propósito de la página para el usuario.

⚽ Sección de Equipos: Grid de confederaciones

Usamos una grid CSS para mostrar las 5 confederaciones en tarjetas automáticas.

<section id="equipos"> <h2>Selecciones participantes</h2> <div class="confed-grid"> <article class="confed"> <h3>África</h3> <ul> <li>Camerún</li> <li>Marruecos</li> <li>Nigeria</li> </ul> </article> <!-- más artículos para Asia, Europa, etc. --> </div> </section>

¿Por qué <article>? Cada confederación es un contenido independiente y reutilizable. ¿Por qué id="equipos"? Para que el enlace del nav pueda saltar directamente aquí.

🏆 Mapa del Torneo: El bracket visual

La parte más compleja: mostrar quién ganó y quién perdió en cada fase. Usamos una estructura de columnas:

<div class="tournament-map"> <div class="round"> <h3>Octavos</h3> <div class="match"> <div class="team-line"> <span class="winner">Rumania</span> <span class="score">3</span> </div> <div class="team-line"> <span class="loser">Argentina</span> <span class="score">2</span> </div> </div> </div> </div>
Lógica visual: La clase .winner aplica color verde y negrita. La clase .loser aplica texto tachado. Las 4 columnas CSS representan: Octavos → Cuartos → Semis → Final.

🎨 CSS: Variables y el sistema de diseño

Al inicio del CSS definimos variables personalizadas (custom properties) para mantener consistencia:

:root { --bg: #0b1020; --card: #161b30; --accent: #00c896; --text: #f4f4f6; --radius: 10px; --space: 1rem; }

Ventajas de usar variables

  • Cambias un color en un solo lugar y se aplica a toda la página.
  • Código más legible y profesional.
  • Fácil de adaptar a modo oscuro/claro.
🍩 Analogía de Homero: Es como tener una receta de donas: si cambias la cantidad de azúcar en la receta, ¡todas las donas cambian!

📐 CSS Grid: El layout inteligente

Para las tarjetas de confederaciones usamos CSS Grid, que distribuye automáticamente los elementos:

.confed-grid { display: grid; grid-template-columns: repeat(auto-fit, minmax(180px, 1fr)); gap: var(--space); }
Traducción: "Crea columnas automáticamente. Cada columna mide mínimo 180px y máximo lo que quepa. El espacio entre columnas es 1rem (16px)."

Para el mapa del torneo usamos un grid de 4 columnas fijas:

.tournament-map { display: grid; grid-template-columns: repeat(4, minmax(0, 1fr)); gap: var(--space); }

🧩 CSS: Componentes visuales

Cada "partido" es una tarjeta estilizada con bordes, sombras y efectos hover:

.match { border-radius: var(--radius); background: radial-gradient(circle at top, rgba(255,255,255,0.02), rgba(7,11,30,0.9)); border: 1px solid var(--border); padding: 0.55rem 0.65rem; transition: all 0.2s ease-out; } .match:hover::after { opacity: 1; /* Borde verde brillante al pasar el mouse */ } .winner { color: var(--accent); font-weight: 600; } .loser { color: var(--muted); text-decoration: line-through; }
✨ Truco de Maggie: El ::after es un pseudo-elemento que crea un borde invisible que solo aparece al hacer hover. ¡Eso da la sensación de que la tarjeta "enciende"!

📱 Responsive: Que se vea en el celular

El diseño debe adaptarse a pantallas pequeñas. Usamos Media Queries:

@media (max-width: 768px) { .slide { padding: 30px 20px; } .two-col { grid-template-columns: 1fr; /* Una sola columna */ } .nav-dots { display: none; /* Ocultar navegación lateral */ } }
📏 ¿Qué hace? Si la pantalla mide 768px o menos (tablets y celulares), el layout de dos columnas se convierte en una sola columna y se ajustan los espaciados.

"En el celular, como en la casa de los Simpson: todo debe caber... aunque sea apretado."

🚀 ¿Qué es GitHub y por qué lo usamos?

GitHub es:

  • Una plataforma de almacenamiento de código en la nube.
  • Un sistema de control de versiones (con Git).
  • Una red social de desarrolladores.
  • Un servidor gratuito para publicar webs estáticas.

GitHub Pages

Servicio incluido que convierte tu repositorio en una página web real con URL del tipo:

tuusuario.github.io/mundial-1994

¡Gratis, sin publicidad y con tu propio dominio!

🎭 Analogía de Krusty: GitHub es tu camerino: guardas tus chistes (código), puedes volver a versiones anteriores si algo falla, y al final subes al escenario (GitHub Pages).

📝 Paso 1: Cuenta y Repositorio

1

Ir a github.com

Entra con tu cuenta. Si no tienes, créala con tu correo (es gratis).

2

Crear nuevo repositorio

Haz clic en el botón + arriba a la derecha → New repository.

3

Configurar nombre

Escribe un nombre descriptivo: mundial-1994 o mi-primera-web.

Selecciona Public (para que GitHub Pages funcione gratis).

NO marques "Add README" si vas a subir tu propio archivo.

4

Crear

Haz clic en Create repository. ¡Listo! Ya tienes tu caja en la nube.

📤 Paso 2: Subir tu archivo HTML

5

Preparar el archivo local

En tu PC, guarda el código en un archivo llamado index.html (¡el nombre importa!). Guarda dentro de una carpeta, por ejemplo: mi-proyecto/.

6

Subir vía web (método más simple)

En tu repositorio de GitHub, haz clic en Add file → Upload files.

Arrastra index.html a la zona de carga o selecciónalo.

7

Confirmar el "commit"

GitHub te pedirá un mensaje. Escribe algo como: "Agregar página Mundial 1994".

Haz clic en Commit changes. Esto guarda la versión en el historial.

8

Verificar

Recarga la página del repositorio. Debes ver index.html listado en la raíz.

⚠️ Importante: El archivo debe llamarse exactamente index.html y estar en la raíz (no dentro de subcarpetas), para que GitHub Pages lo reconozca automáticamente como página de inicio.

🌐 Paso 3: Activar GitHub Pages

9

Ir a Settings

En tu repositorio, haz clic en la pestaña Settings (la última, arriba).

10

Buscar la sección Pages

En el menú lateral izquierdo, busca y haz clic en Pages.

11

Seleccionar la fuente

En "Source" o "Branch", selecciona main y carpeta / (root).

Haz clic en Save.

12

Esperar y copiar la URL

GitHub construye tu sitio en segundos. Aparecerá un mensaje verde con la URL:

https://tu-usuario.github.io/mundial-1994/

🎉 ¡Woo-hoo! Tu página ya está en internet! Puedes compartir ese enlace con cualquier persona del mundo.

✅ Paso 4: Verificar y compartir

13

Abrir la URL

Copia la URL de GitHub Pages, pégala en tu navegador y verifica que la página se ve correctamente.

14

Probar en el celular

Abre la URL en tu celular. Si aplicaste responsive design, debe verse bien también en pantalla pequeña.

15

Compartir

Envía el enlace por WhatsApp, pégalo en el foro de la clase, o inclúyelo en tu portafolio.

16

Actualizar (bonus)

Si mejoras la página, sube el archivo de nuevo con el mismo nombre. GitHub Pages se actualizará automáticamente en pocos minutos.

🍺 Consejo de Barney: Si algo no se ve bien, presiona Ctrl+F5 (Windows) o Cmd+Shift+R (Mac) para forzar recarga sin caché.

🎯 Reto para los aprendices

El verdadero aprendizaje está en hacerlo ustedes mismos. Su misión:

🍩 El Desafío Simpson

  1. Piensen en un tema que les apasione (música, videojuegos, comida, deportes, Los Simpson...).

  4. Personalicen algo: cambien un color, un texto o una imagen.
  5. Suban la página a GitHub siguiendo los 16 pasos que acabamos de ver.
  6. Compartan la URL en el chat de la clase.
📋 Entregable: Enviar por la plataforma: (1) el enlace de GitHub Pages funcionando, y (2) una captura de pantalla del código mostrando al menos 3 etiquetas HTML que identificaron.
