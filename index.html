🧠 Paso 0: El Prompt (La receta)

Antes de tocar código, le pedimos a la IA que cree la página. Un buen prompt incluye:

"Ayúdame a crear una página web sobre la Copa Mundial de Fútbol 1994 que contenga: - Todos los equipos participantes - Un mapa conceptual del torneo (quién ganó y perdió) - Usar HTML y CSS puro - Que sea visual y fácil de entender"
💡 Tip de Bart: Mientras más específico seas, mejor resultado obtienes. Incluye formato, colores, datos específicos y tipo de interacción.

La IA genera el código HTML completo con estilos CSS embebidos. ¡No necesitas instalar nada!

📄 La anatomía de un archivo HTML

Todo documento HTML tiene una estructura básica obligatoria:

<!DOCTYPE html> <html lang="es"> <head> <meta charset="UTF-8"> <title>Mi Página</title> <style>...CSS aquí...</style> </head> <body> <h1>Hola Springfield</h1> </body> </html>
<head>: El "cerebro" invisible. Contiene título, estilos, fuentes y configuración.
<body>: El "cuerpo" visible. Todo lo que el usuario ve y interactúa.

🏗️ Etiquetas semánticas: La estructura lógica

HTML5 introdujo etiquetas que describen su propósito:

<header> <!-- Encabezado principal: título, nav --> <nav> <!-- Navegación: enlaces internos --> <main> <!-- Contenido principal único de la página --> <section> <!-- Sección temática: equipos, mapa --> <article> <!-- Artículo independiente: cada confederación --> <div> <!-- Contenedor genérico para agrupar --> <h1> a <h3> <!-- Jerarquía de títulos --> <p> <!-- Párrafos de texto --> <ul> / <li> <!-- Listas desordenadas y items -->
🎭 Como diría Lisa: Usar etiquetas semánticas no es solo "ser correcto", ayuda a que lectores de pantalla, motores de búsqueda y tu futuro yo entiendan la estructura.

🎨 CSS: Variables y el sistema de diseño

Al inicio del CSS definimos variables personalizadas para mantener consistencia:

:root { --bg: #0b1020; --card: #161b30; --accent: #00c896; --text: #f4f4f6; --radius: 10px; --space: 1rem; }

Ventajas de usar variables

  • Cambias un color en un solo lugar y se aplica a toda la página.
  • Código más legible y profesional.
  • Fácil de adaptar a modo oscuro/claro.
🍩 Analogía de Homero: Es como tener una receta de donas: si cambias la cantidad de azúcar en la receta, ¡todas las donas cambian!

📱 Responsive: Que se vea en el celular

El diseño debe adaptarse a pantallas pequeñas. Usamos Media Queries:

@media (max-width: 768px) { .slide { padding: 30px 20px; } .two-col { grid-template-columns: 1fr; } .nav-dots { display: none; } }
📏 ¿Qué hace? Si la pantalla mide 768px o menos (tablets y celulares), el layout de dos columnas se convierte en una sola columna y se ajustan los espaciados.

"En el celular, como en la casa de los Simpson: todo debe caber... aunque sea apretado."

🚀 GitHub Pages: Tu página en internet

¿Qué es GitHub Pages?

Es un servicio GRATUITO que convierte tu repositorio en una página web real con URL propia.

✅ Ventajas:
  • Totalmente gratis (sin publicidad)
  • Tu propio dominio con tu usuario de GitHub
  • Se actualiza automáticamente cuando subes cambios
  • Accesible desde cualquier dispositivo
  • ¡Todo el mundo puede abrir el link!
